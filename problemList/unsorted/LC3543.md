```java
public class LC3543_TLE {
    private int ret = -1;
    private List<int[]>[] g;
    private int k, t;
    public int maxWeight(int n, int[][] edges, int _k, int _t) {
        this.k = _k;
        this.t = _t;
        g = new List[n];
        for(int i = 0;i < n;i++) g[i] = new ArrayList<>();
        for(int[] e : edges){
            g[e[0]].add(new int[]{e[1], e[2]});
        }
        for(int i = 0;i < n;i++){
            dfs(i, 0, 0);
        }
        return ret;
    }

    // 当前节点编号, 当前走过的边数, 当前路径上的边权和
    private void dfs(int i, int curCnt, int sum){
        if(curCnt == k){
            if(sum < t) ret = Math.max(sum, ret);
            return;
        }
        for(int[] nxt : g[i]){
            if(sum + nxt[1] < t){
                dfs(nxt[0], curCnt + 1, sum + nxt[1]);
            }
        }
    }
}
```

暴力dfs的时间复杂度 (分析来自deepseek): 

上述代码的时间复杂度为 **O(n \* d^k)**，其中：

- **n** 是节点总数。
- **d** 是图中节点的最大出度（即每个节点最多的直接后继边数）。
- **k** 是路径所需的边数。

关键分析步骤：

1. **递归深度**：DFS的深度为 `k`，因为每次递归增加一条边，直到恰好 `k` 条边。
2. **分支因子**：每次递归遍历当前节点的所有出边，假设最大出度为 `d`，则每层递归最多有 `d` 个分支。
3. **起点数量**：外层循环对每个节点作为起点调用DFS，共 `n` 次。
4. **总路径数**：每个起点的DFS最多探索 `d^k` 条路径（指数级增长），总时间复杂度为 `n * d^k`。
