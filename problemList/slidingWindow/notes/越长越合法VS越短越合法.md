### 滑窗中 越长越合法 与 越短越合法 的区别和联系

首先我们需要明白滑窗的原理

滑窗是对暴力枚举区间左右端点的优化, 在暴力做法中, 我们要分别枚举区间的左右端点, 以此来确定区间范围

而滑动窗口的引入, 就是利用了这个区间的某些特性 (比如越短越合法, 或越长越合法) 从而优化其中的某一个指针的移动, 使得这个指针不再回退, 因此将时间复杂度从暴力枚举左右端点的`O(n ^ 2)`优化到`O(n)`

### 什么是 越长越合法? 什么是 越短越合法? 

这就涉及到可以使用滑窗优化的区间所具有的特点

1. 越长越合法: 对于固定了一个端点的区间来说, 如果这个区间长度不超过某个固定长度时, 这些区间都是非法的, 而当区间长度超过了某个固定长度后, 这些区间都是合法的

    具有 越长越合法 特性的区间, 一般我们会求这个区间的**最短长度**

2. 越短越合法: 和上面类似, 当区间长度大于某个固定长度, 这些区间都是非法, 当小于某个区间长度, 这些区间都是合法

    类似的, 具有 越短越合法 特性的区间, 一般我们会求这个区间的**最长长度**

#### 越短越合法 的区间如何滑窗?

这是比较常见的情况, 也就是题单中2.1小节部分

我们不妨从`O(n ^ 2)`的暴力枚举来推导出这种情况下滑窗的写法

首先考虑暴力情况, 暴力情况下, 我们需要枚举每个区间的左右端点, 然后判断当前区间是否合法

但是在这种情况下, 我们枚举的区间具有 **越长越合法** 的特性

那么我们如何利用这个特性来优化暴力中两个区间端点指针的移动呢?

这里不妨假设枚举区间端点的两个指针分别叫`left`和`right`

由于区间具有 越短越合法 的特性, 因此对于`right`固定的这些区间来说, `left`在从左到右枚举的过程中, 一开始都是非法区间, 后来都是合法区间

假设我们暴力枚举到了`[left, right]`这样的一个区间, 这个区间是第一个合法区间, 那么考虑接下来我们要干什么

首先根据区间特性, 我们知道, 在右端点为`right`的这些区间中, 左端点在当前`left`右边的这些所有的区间, 都是合法的, 因此我们就知道了, 右端点为`right`的这些区间中, 最长区间长度就是当前的`right - left + 1`

因此右端点为`right`的这些区间就都枚举完了, 接下来`right++`, 枚举下一个右端点

关键的来了, 这里我们假设`right++`之前的`right = idxRight`, 那么`right++`之后`right = idxRight + 1`, 并且假设之前的`left = idxLeft`, 即我们认为`left`和`right`是移动的两个指针(这两个指针的值是会变的), 而`idxLeft`和`idxRight`是两个固定值, 其等于刚才找到的以`right`为区间右端点的区间中最长的那个合法区间的左右端点

对于此时右端点为`idxRight + 1`的这些区间来说, 在暴力做法中, 我们是需要让`left`指针从下标`0`重新开始向右移动, 但是这里我们可以利用区间特性来缩小`left`指针的移动范围

在枚举区间右端点为`idxRight`的这些区间时, 我们知道, `[idxLeft, idxRight]`是最长的一个合法区间, 也就是在满足`0 <= i < idxLeft`的情况下, `[i, idxRight]`的这些区间都是非法的

那么对于在`0 <= i < idxLeft`范围内的每一个`i`, 我们将`[i, idxRight]`的区间右端点向右移动一位, 移动到`[i, idxRight + 1]`, 也就是现在右端点指针指向的`right = idxRight + 1`的位置

对于`[i, idxRight + 1]`的这些区间来说, 对于每一个可能的`i`, 我们将这个区间看成是由`[i, idxRight]`右端点`+1`得到的`[i, idxRight + 1]`, 根据区间特性, 在某个端点确定(左端点`i`确定)的情况下, 一旦区间长度超过了某一个值, 那么再长的区间都是非法, 那么此时由于`[i, idxRight]`这个区间是非法, 那么更长的`[i, idxRight + 1]`区间一定是非法

**因此我们得到:** 对于满足`0 <= i < idxLeft`的这些`i`, 其对应的`[i, idxRight + 1]`(或者说`[i, right]`)这些区间, 都是非法的

因此在枚举右端点`right == idxRight + 1`的这些区间中, 我们就没有必要让左端点指针`left`从`0`下标开始移动了, 而是可以直接从`idxLeft`开始向右移动, 继续遍历接下来更短的这些区间是否合法

通过上面的分析我们可以看到, 左端点指针`left`是 **不减** 的, 因此从这里也可以看出滑窗的时间复杂度是`O(n)`的

#### 越短越合法 的滑窗代码如何编写?

在滑窗中, 适当的引入一些模版, 可以极大的加快思考过程, 以及编码中可能出现的错误

要想写 越短越合法 的滑窗代码, 首先我们再回顾一下 越短越合法 的滑窗中, 区间左右端点的移动规律

1. 右端点依次枚举`[0, n - 1]`中的所有下标

2. 对于某个`right == i`的区间来说, 开始移动左指针`left`, 移动左指针`left`的前提是: **当前区间要是一个合法区间**, 如果当前区间是一个合法区间, 那么就循环移动左指针`left`, 尝试找到一个更短的合法区间

3. 一直移动左指针`left`直到找到第一个不合法的区间, 那么此时停止移动左指针`left`, 将右指针`right + 1`, 继续枚举右端点为`right + 1`的这些区间

因此伪代码如下: 

```java
// 这里循环中的i就是右端点
for(int i = 0;i < n;i++){
    // 维护某些信息...
    while(区间合法){    // 只要区间合法, 就移动左指针
        left++;
        // 维护某些信息...
    }
    // while循环退出时, 此时的[left, i]区间, 就是当前右端点为i的情况下, 最长的不合法区间
    // 更新某些信息...
}
```
