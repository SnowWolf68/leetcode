Dijkstra求次短路 (https://leetcode.cn/problems/second-minimum-time-to-reach-destination/description/)

原理: 
先回顾一下dijkstra的原理: 通过每次选取一个已经确定最小距离的点, 并用这个点去松弛到其余点的最小距离, 最终得到要求的两点之间的最短距离. 其中每次在dist中选min一定是新确定的最小距离的点, 该结论的正确性可以用反证法证明

如何将上述原理推广到求次短路? 一种朴素的想法是: 求最短路时我们用的是已经确定最短路的点拓展, 那么类似的, 如果用已经确定次短距离的点拓展, 那么拓展出来的不就是次短路了?

这种想法有点道理, 但是这样拓展出来的点不能包括全部的次短路

因为次短路也有可能通过某一段最短路拼接上一段次短路得到, 即 0 -> 5 的次短路, 可以有 0 -> 3 的最短路 + 3 -> 5 的次短路 得到

因此不难得出正确的更新方式: 首先肯定要同时拓展最短路和次短路, 因此需要两个dist数组dist1, dist2分别记录最短巨鹿, 次短距离. 拓展最短距离时, 如果当前拓展的距离能够更新dist1, 那么说明找到最短距离, 如果不能更新dist1, 并且当前这个距离不等于dist1中对应点的最短距离时(这里保证找到的是严格次短, 如果不要求严格次短, 那么不需要这个判断), 那么可以更新dist2. 拓展次短距离时, 如果当前拓展到的距离能更新dist2, 则更新, 否则不更新

值得注意的是, 如果要求严格次短, 那么dist1[startNode]可以初始化为0, 但是dist2[startNode]不能初始化为0, 要初始化为INF. 原因也很显然, 严格次短显然不能和最短相等, 否则不叫严格次短

具体实现: 
这里分别说一下朴素dijkstra和堆优化dijkstra的实现

松弛次数?
在实现朴素dijkstra时, 首先要确定的一点是松弛次数
在最短路中, 需要n - 1次松弛, 因为每次松弛都要选一个已经确定最短距离的点, 用这个点来松弛, 并且最后选出来的最短距离的点不需要再次松弛, 因此总共的松弛次数为n - 1次

如果问题变成了求次短路, 会怎么样?
不妨把松弛最短和松弛次短分开考虑, 在原理部分已经分析得到, 松弛最短时, 需要同时更新dist1和dist2, 但是松弛次短时, 只需要更新dist2. 那么可以先考虑松弛最短, 此时要n次松弛. 为什么要在原来n - 1的基础上加一次松弛次数? 因为最后的一次松弛虽然不会更新dist1, 但是有可能更新dist2. 然后考虑松弛次短, 由于这里最后一次得到的次短点就不需要再松弛了, 因此是n - 1次. 总的松弛次数就是 2 * n - 1 次
(从上面的分析也可以更好的理解为什么示例2 (n = 2, edges = [[1,2]], time = 3, change = 2) 的次短路线是怎么走出来的)

这里的松弛次数 2 * n - 1 会在朴素dijkstra中体现出来

朴素dijkstra实现: 
在实现中, 我们可以将松弛最短和松弛次短合在一起做, 具体来说, 即在一开始的 "挑选已经确定距离的最短点" 时, 将最短和次短都挑出来. 但是这里需要注意, 我们一定要保证, 挑出来的minDist是dist1中的最小值, secondDist是 严格大于minDist 的, dist1和dist2中的次小值, 即挑选出来的两个min都要是符合要求的全局最小值 (这里的错误写法见LC2045_3_TLE中注释掉的部分)

在更新dist时, 就是严格按照之前提到的更新规则进行: 使用minDist更新dist1和dist2, 使用secondDist更新dist2

(朴素dijkstra的正确写法见 LC2045_4_TLE)

堆优化dijkstra实现: 
堆优化这里就比较简单, 由于这里我们也不需要考虑松弛次数, 只要堆不空, 就从堆里弹一个出来松弛即可
松弛最短和次短的规则和原理中分析的一样, 只不过别忘了更新完dist2之后, 也要将对应的点入堆, 因为后续可能还需要用这个次短距离的点来松弛其他点

(堆优化dijkstra的正确写法见LC2045_2)
