dp总结

Motivation: 虽然做了很多的dp, 但是过段时间看看之前的问题还是没有思路, 因此决定对dp问题做一个总结, 总结一些自己的东西出来

PART 1: 背包dp

[LC871](https://leetcode.cn/problems/minimum-number-of-refueling-stops/description/)

在我们走到target的过程中, 会走过沿途的所有加油站, 对于每一个加油站, 此时我们都有两种选择, 1)加油 2)不加油  又由于背包dp的核心是 "选或不选" , 那么我们可以尝试用背包dp来解决这个问题

1. 考虑状态定义: 首先根据背包dp的要求, 肯定要有一个维度`i`来表示当前考虑`[0, i]`区间的元素, 除此之外, 由于本题问的是 "最低加油次数" , 那么显然还需要一个维度`j`来表示 "当前加了多少次油" , 最后是`dp[i][j]`的值的含义, 我们可以将其定义为能够走到的最远距离, 因此最终的状态表示我们定义为:  

   `dp[i][j]`表示考虑下标在`[0, i]`区间的加油站, 加了`j`次油, **能够走到的最远距离**

2. 接着考虑状态转移方程: 对于下标为`i`的加油站, 此时我们有两种选择 (这也是背包dp的经典套路): 加油 或 不加油

   1. 加油: 要想在下标为`i`的加油站加油, 首先需要满足条件: 能够走到下标为`i`的加油站, 即满足 

      `dp[i - 1][j - 1] >= stations[i][0]`

      如果能够满足, 那么我们就可以在`dp[i - 1][j - 1]`的基础上多跑`stations[i][1]`这么远, 即

      `dp[i][j] = dp[i - 1][j - 1] + stations[i][1];`

   2. 不加油: 此时问题转化为: 考虑`[0, i - 1]`区间的加油站, 加`j`次油, 能够走到的最远距离, 即此时有

      `dp[i][j] = dp[i - 1][j];`

​	对于上面两种情况, 我们此时显然需要取一个最大值(max)

3. 考虑初始化: 由于这里`i - 1, j - 1`都有可能越界, 因此需要考虑初始化问题. 一般来说这里都是选择添加一行一列辅助节点, 但是由于这里列表示: 加了`j`次油. 因此显然添加的一列辅助节点没有实际含义, 因此最终我选择只添加一行辅助节点, 其中第一行表示 不考虑任何的加油站

​	如何初始化? 由于第一行表示不考虑任何加油站, 那么除了`dp[0][0]`这个位置, 其余位置都是非法, 因为不考虑任何加油站的情况下只能加`0`次油, 因此初始化第一行(除去`dp[0][0]`)为`-INF`, 第一列表示加了`0`次油, 因此初始化第一列为`startFuel`

> 有关这里非法位置到底是初始化为`0`还是初始化为`-INF`的问题: 
>
> 结论: 都可以
>
> 解释: 假设这里我们初始化成`0`, 那么就会导致后面的某些状态, 可能会依赖到这里的"不考虑任何加油站, 并且加了`j (j > 0)`次油"的状态, 但是即使依赖于这个状态, 也不会导致后续更新的错误. 因为如果依赖了这个状态, 那么意味着其相比于依赖 `dp[0][0]` (即不考虑任何加油站, 并且加了`0`次油)  的状态少加了若干次油, 那么显然其能够走到的最远距离就要比其依赖于 `dp[0][0]` 要近, 那么这种依赖情况显然不会是最优解, 因此不会影响整个dp表的正确更新

4. 考虑填表顺序: 从上往下, 从左到右填表即可
5. 返回值: 本题问的是到达目的地的最少加油次数, 那么我们从小到大可以遍历`dp`表中的`dp[n][i]`这些列, 找到第一个满足`dp[n][i] >= target`的`i`, 即为最少加油次数

需要注意的是, 这里在添加了一行辅助节点的情况下, 状态转移中的下标需要作一下变换: 

`dp[i - 1][j - 1] >= stations[i][0] --> dp[i - 1][j - 1] >= stations[i - 1][0]`

`dp[i][j] = dp[i - 1][j - 1] + stations[i][1] --> dp[i][j] = dp[i - 1][j - 1] + stations[i - 1][1]`

```java
public int minRefuelStops(int target, int startFuel, int[][] stations) {
      int n = stations.length, INF = 0x3f3f3f3f;
      int[][] dp = new int[n + 1][n + 1];
      Arrays.fill(dp[0], -INF);		// 不写这行也可以
      for(int i = 0;i <= n;i++) dp[i][0] = startFuel;
      for(int i = 1;i <= n;i++){
          for(int j = 1;j <= n;j++){
              dp[i][j] = dp[i - 1][j];
              if(dp[i - 1][j - 1] >= stations[i - 1][0]){
                  dp[i][j] = Math.max(dp[i][j], dp[i - 1][j - 1] + stations[i - 1][1]);
              }
          }
      }
      for(int i = 0;i <= n;i++){
          if(dp[n][i] >= target) return i;
      }
      return -1;
  }
```

[LC2915](https://leetcode.cn/problems/length-of-the-longest-subsequence-that-sums-to-target/description/  )

对于每一个元素, 我们都有 "选或不选" 两种可能, 因此可以使用背包dp

我们可以仿照上一题中设计状态表示的经验, 给出这题的状态表示: 

1. `dp[i][j]` 表示考虑`[0, i]`区间的元素, 组成和为`j`的子序列, 此时子序列的长度最大值

2. `dp[i][j]`: 对于下标为`i`的这个元素, 此时我们有两种选择: 选或不选

   1. 选: 那么问题转化为 从`[0, i - 1]`的区间中选出和为`j - nums[i]`的子序列, 此时的最长长度, 即有

      `dp[i][j] = dp[i - 1][j - nums[i]] + 1;`

   2. 不选: 那么问题转化为 从`[0, i - 1]`中选出和为`j`的子序列, 此时的子序列的最长长度, 即有

      `dp[i][j] = dp[i - 1][j];`

   上面两者我们显然需要取一个max

3. 初始化: 这里`i - 1, j - nums[i]`都有可能越界, 因此需要考虑初始化. 

   对于`i - 1`的越界问题, 此时我们可以添加一行辅助节点, 此时第一行意味着 不考虑任何元素 , 那么显然不考虑任何元素时, 子序列的和只能是`0`, 因此`dp[0][0] = 0`, 其余位置都是非法 (不考虑任何元素时, 子序列的和非`0`), 因此初始化为`-INF`

   > 这里需要注意, 和第一题不同的是, 这里的非法位置我们只能初始化为`-INF`, 而不能初始化为`0`
   >
   > 不妨这样考虑: 假设后面某一个`dp[i][j]`依赖到了`dp[0][i] = 0`的状态, 那么此时这个`dp[0][i] = 0`能不能保证`dp[i][j]`更新正确? 
   >
   > 对于第一题来说, 即使依赖到`dp[0][i] = 0`的状态, 也就相当于在整个过程中相比依赖于`dp[0][0]`, 加油的次数变少了, 因此依赖与`dp[0][i]`的情况一定不会是`dp[i][j]`的最优解, 因此没问题
   >
   > 但是对于这题来说, 如果`dp[i][j]`依赖了`dp[0][i] = 0`, 那么就相当于`dp[i][j]`依赖了一个 **没有任何元素, 但是和为`i`(不为0), 并且长度等于`0` 的序列** , 这显然是不行的

   对于`j - nums[i]`的越界问题, 由于`nums[i]`不总是`1`, 那么添加辅助节点的方法就不行了, 但是我们可以考虑一下实际意义, 如果`j - nums[i] < 0`的话, 也就是说要在`[0, i - 1]`区间内, 找出一个和为负数的子序列, 这显然是不行的, 那么此时我们可以直接不考虑这种情况即可

   **这里有一个易错点, 就是虽然`j - nums[i]`的越界我们在状态转移中手动控制, 但是我们还是需要处理第一列的初始化的**

   对于第一列来说, 此时意味着考虑`[0, i]`区间的元素, 组成和为`0`的子序列, 那么显然子序列长度只能是`0`, 因此初始化第一列全都为`0`

4. 填表顺序: 从上往下, 从左往右即可

5. `return dp[n][target] < 0 ? -1 : dp[n][target];`

   >  注: 这里对于不能组成`target`的情况应该用`< 0`来判断, 而不能用`== -INF`来判断, 这其实也很好理解, 因为状态转移方程`dp[i][j] = dp[i - 1][j - nums[i]] + 1`中依赖的`dp[i - 1][j - nums[i]]`这个状态也可能是非法值, 即`-INF`, 因此`dp[i][j]`就有可能是`-INF + 1`, 从而使得`dp[i][j] > -INF`, 但是仍然满足`dp[i][j] < 0`


注: 添加辅助节点引起的下标映射问题, 可以类似第一题的处理方式

```java
public int lengthOfLongestSubsequence(List<Integer> nums, int target) {
      int n = nums.size(), INF = 0x3f3f3f3f;
      int[][] dp = new int[n + 1][target + 1];
      for(int i = 1;i <= target;i++) dp[0][i] = -INF;
      for(int i = 1;i <= n;i++){
          for(int j = 1;j <= target;j++){
              dp[i][j] = dp[i - 1][j];
              if(j - nums.get(i - 1) >= 0) dp[i][j] = Math.max(dp[i][j], dp[i - 1][j - nums.get(i - 1)] + 1);
          }
      }
      return dp[n][target] < 0 ? -1 : dp[n][target];
  }
```

背包dp的优化 -- 滚动数组优化

原理: 在背包dp中, 由于`dp[i][j]`只会依赖与上一行`dp[i - 1]`中的某个元素, 因此我们可以将二维的dp表简化为一维的数组

需要注意的是, 在简化为数组之后, 我们就需要考虑**覆盖**的问题, 即新计算出来的`dp`值会覆盖掉旧的`dp`值, 使得其他的依赖与这个位置的dp值的`dp[j]`就都会计算错误, 因此我们需要调整遍历方式, 使得当某个`dp[j]`被计算出来之后, 其他位置就不再会依赖于这个位置

由于在状态转移中, 我们都是依赖于当前位置或当前位置前面某列的`dp[j]`, 因此不难看出, 如果我们将`j`的遍历顺序变成 **从大到小** , 那么就能保证 只要`dp[j]`被计算出来之后, 其余位置的`dp`值的计算不会依赖于这个`dp[j]`

```java
public int lengthOfLongestSubsequence(List<Integer> nums, int target) {
      int n = nums.size(), INF = 0x3f3f3f3f;
      int[] dp = new int[target + 1];
      Arrays.fill(dp, -INF);
      dp[0] = 0;
      for(int num : nums){		// 使用这种遍历方式, 能够大大加快运行速度
          for(int j = target;j >= 1;j--){
              if(j - num >= 0) dp[j] = Math.max(dp[j], dp[j - num] + 1);
          }
      }
      return dp[target] < 0 ? -1 : dp[target];
  }
```
