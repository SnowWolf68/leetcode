一开始我的代码是这样的: 

```java
public double maxProbability(int n, int[][] edges, double[] succProb, int start_node, int end_node) {
    List<double[]>[] g = new List[n];
    for(int i = 0;i < n;i++) g[i] = new ArrayList<>();
    for(int i = 0;i < succProb.length;i++){
        g[edges[i][0]].add(new double[]{edges[i][1], succProb[i]});
        g[edges[i][1]].add(new double[]{edges[i][0], succProb[i]});
    }
    double[] dist = new double[n];
    dist[start_node] = 1;
    PriorityQueue<double[]> minHeap = new PriorityQueue<>((a, b) -> Double.compare(a[1], b[1]));
    minHeap.offer(new double[]{start_node, 1});
    // boolean[] done = new boolean[n];
    while (!minHeap.isEmpty()) {
        double[] poll = minHeap.poll();
        int cur = (int)poll[0];
        // if(done[cur]) continue;
        // done[cur] = true;
        for(double[] nxtE : g[cur]){
            int nxt = (int)nxtE[0];
            double prob = nxtE[1];
            if(dist[cur] * prob > dist[nxt]){
                dist[nxt] = dist[cur] * prob;
                minHeap.offer(new double[]{nxt, dist[nxt]});
            }
        }
    }
    return dist[end_node];
}
```

此时能通过部分用例, 不过超时了, 但是当我加上done数组, 即上面注释掉的这几行之后, 直接WA了

于是发现这里应该用大根堆, 而我用了小根堆

但是之前为什么也能够通过部分用例呢? 

这是因为由于我没有加done的判断, 导致同一个节点可能会被处理多次, 而由于每一次处理某一个点都会将其所有的邻接点入堆, 因此能够保证正确性

但是显然时间复杂度会变高

deepseek的回答如下: 

该实现通过牺牲部分效率（允许重复入队）换取了正确性，本质上将Dijkstra算法改造成了类似广度优先搜索的动态规划。虽然能通过所有测试用例，但在稠密图上的时间复杂度可能恶化到O(E²)。建议仍采用最大堆实现以获得最优性能。
