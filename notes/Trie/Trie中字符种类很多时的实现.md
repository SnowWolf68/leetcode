### Trie中字符种类很多时的实现

在静态数组实现方式中, 由于我们使用了静态数组, 因此就不能再用哈希表来存储所有孩子节点, 那么此时如果Trie中的字符种类很多, 那应该怎么处理?

如果Trie中的路很多, 比如其中一条路是`1729`, 那么此时如果我们还是按照之前的实现方式, 那么显然`tree[][]`数组的第二个维度需要至少`1729`长度

如果路更多, 显然`tree`数组的第二个维度所需要的长度就更长, 就有可能开不出来这么长的空间

此时我们可以将这条路拆解, 即将`1729`拆解成`1`, `7`, `2`, `9`, `#`, 这5层的路

```
原来的情况: 
      node
     /    \
    /      \ (1729这条路)
   /        \
           subNode 

后来的情况
        node
            \ (1这条路)
            subNode1
                \ (7)这条路
               subNode2 
                    \   (2这条路)
                    subNode3
                        \   (9这条路)
                        subNode4
                            \   (#这条路)
                           subNode5
```

在进行这样的修改之后, 每个节点最多的子节点个数就变得有限了 (如果路径只用数字表示的话, 那么一个节点的子节点至多就是 `10 + 1 = 11` 个 (10个数字 + 1个`#`字符))

需要注意的是, 这里之所以要在原本的`1729`这条路后面添加上一个`#`, 目的是为了区分不同的路, 换句话说, 就是为了区分一条路的结束位置

如果还是不大明白, 可以考虑这个例子, 此时有`1729, 17`两条路, 如果不添加`#`这个符号的话, 那么展开之后, 如果我们遍历到`1`和`7`之后, 就无法区分这条路到底是`1729`还是`17`, 会发生混淆
