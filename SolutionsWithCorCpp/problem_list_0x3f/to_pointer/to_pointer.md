### 为什么双指针不会漏掉某些情况?

[167. 两数之和 II - 输入有序数组](https://leetcode.cn/problems/two-sum-ii-input-array-is-sorted/)

> 给你一个下标从 **1** 开始的整数数组 `numbers` ，该数组已按 **非递减顺序排列** ，请你从数组中找出满足相加之和等于目标数 `target` 的两个数。如果设这两个数分别是 `numbers[index1]` 和 `numbers[index2]` ，则 `1 <= index1 < index2 <= numbers.length` 。
>
> 以长度为 2 的整数数组 `[index1, index2]` 的形式返回这两个整数的下标 `index1` 和 `index2`。
>
> 你可以假设每个输入 **只对应唯一的答案** ，而且你 **不可以** 重复使用相同的元素。
>
> 你所设计的解决方案必须只使用常量级的额外空间。
>
> > **示例 1：**
> >
> > ```
> > 输入：numbers = [2,7,11,15], target = 9
> > 输出：[1,2]
> > 解释：2 与 7 之和等于目标数 9 。因此 index1 = 1, index2 = 2 。返回 [1, 2] 。
> > ```
> >
> > **示例 2：**
> >
> > ```
> > 输入：numbers = [2,3,4], target = 6
> > 输出：[1,3]
> > 解释：2 与 4 之和等于目标数 6 。因此 index1 = 1, index2 = 3 。返回 [1, 3] 。
> > ```
> >
> > **示例 3：**
> >
> > ```
> > 输入：numbers = [-1,0], target = -1
> > 输出：[1,2]
> > 解释：-1 与 0 之和等于目标数 -1 。因此 index1 = 1, index2 = 2 。返回 [1, 2] 。
> > ```

首先想到的暴力解法就是两层循环遍历每一个子区间的左右端点, 时间复杂度为`O(n ^ 2)`级别

如果我们把暴力解法稍微调整一下, 即原本的暴力解法中的循环方向为:

```cc
for(int i = 0;i < n;i++){
    for(int j = i;j < n;j++){
        ...
    }
}
```

将内层的`j`的循环方向倒过来, 变成:

```cc
for(int i = 0;i < n;i++){
    for(int j = n - 1;j >= i;j--){
        ...
    }
}
```

接下来我们考虑如何根据 **原数组有序** 的条件, 将暴力解法**优化**为双指针

考虑示例: `nums = [1, 4, 5, 8, 9], target = 11`

首先考虑`i == 0, j == 4`即`nums[i] == 1, nums[j] == 9`的情况
此时由于`1 + 9 == 10 < 11`, 因此`[1, 9]`不符合要求. 又由于原数组有序, 那么1和任何比9小的数都不可能符合要求, 因此让`i++`

此时两个端点来到了`[4, 9]`, 由于`4 + 9 == 13 > 11`, 因此`[4, 9]`不符合要求. 由于原数组有序, 那么9和所有比4大的都不可能符合要求, 因此`j--`

此时两个端点来到了`[4, 8]`, 由于`4 + 8 == 12 > 11`, 因此`[4, 8]`不符合要求. 由于原数组有序, 那么8和所有比4大的数都不可能符合要求

+ 此时问题来了, 虽然此时我们能够得到 "8和所有比4大的数都不符合要求" , 那么 **8和比4小的数有没有可能符合要求呢?**

很多讲解并没有解释这一点, 下面我详细解释一下

首先需要明白一点, 既然我们枚举的是两个端点, 那么这两个端点的所有取值就构成了一个矩阵(二维表). 又由于这里我们枚举的两个端点满足`i <= j`, 那么实际上就是一个矩阵的上三角部分, 如下

右上角的空白部分就是本题中所有可能的`[i, j]`取值 (包括主对角线)

如果我们在图上将之前的移动过程描述出来, 就是这样: 

<img src="C:\Users\zhang\Desktop\dev\github\leetcode\SolutionsWithCorCpp\problem_list_0x3f\to_pointer\img_to_pointer\image-20260313172123917.png" alt="image-20260313172123917" style="zoom: 50%;" />

此时我们可以发现几个规律: 

1. 格子的起始位置为右上角, 即下标`[0, n - 1]`处
2. (没有找到`target`的前提下)
   + 每次移动一次, 都只能是在原格子的基础上, **向下**或**向左**移动.
   + 每次移动, 都能排除**一整行**或**一整列**的非法格子

通过以上两点, 加上图示的描述, 我们不难得到如下规律:

+ 对于某一次的`[i, j] = [x, y]`来说, 此时所有可能符合要求的区间只有`i >= x, j <= y`, 对于其他区间的`[i, j]`, 都一定不能符合要求

那么此时最初的问题就迎刃而解了

> 问题回顾: `i == 1, j == 4`即`[nums[i], nums[j]] == [4, 8]`时, 通过`4 + 8 = 12 > 11`, 虽然此时我们能够得到 "8和所有比4大的数都不符合要求" , 那么 **8和比4小的数有没有可能符合要求呢?**

对于`i == 1, j == 4`即`[nums[i], nums[j]] == [4, 8]`时, 此时若令`nums[j] == 8`不动, 并且让`nums[i] < 4`的话, 在二维表格中对应的这些格子**已经**在之前的搜索过程中**被排除**了!

<img src="C:\Users\zhang\Desktop\dev\github\leetcode\SolutionsWithCorCpp\problem_list_0x3f\to_pointer\img_to_pointer\image-20260313175446312.png" alt="image-20260313175446312" style="zoom: 50%;" />

因此让两个指针`i, j`单向移动的正确性就能够保证, 因此原本的**暴力循环**就变成了**相向双指针**!
